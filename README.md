## Branch Naming Convention

Feature branches follow the naming convention:

```text
week-<week-number>-<topic-name>
```

Example:

```text
week-01-git-branching
```
Each week's branch contains practical exercises and notes related to the topics covered during that phase of the training.
---

## Contribution Workflow

1. Create a feature branch.
2. Implement the assigned task.
3. Commit changes with meaningful commit messages.
4. Push the branch to GitHub.
5. Create a Pull Request.
6. Review and merge after approval.
